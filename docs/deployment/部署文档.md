# 部署文档

## 1. 环境要求

### 1.1 系统要求

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| CPU | 2 vCPU | 2-4 vCPU |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 20 GB | 50 GB |
| 操作系统 | Ubuntu 22.04+ / Debian 11+ | Ubuntu 22.04+ / Debian 12+ |
| Docker | 20.10+ | 最新版 |
| Docker Compose | 2.0+ | 最新版 |

### 1.2 支持的操作系统

- Ubuntu 22.04 LTS 及以上
- Debian 11 (Bullseye) 及以上
- 其他 Linux 发行版 (需自行安装 Docker)

## 2. 部署方式

### 2.1 一键部署 (推荐)

适用于全新的 Ubuntu/Debian 服务器。

```bash
curl -fsSL https://raw.githubusercontent.com/haoyiyin/basjoo/main/install-deploy.sh | sudo sh
```

**安装脚本功能**:
1. 自动检测并安装 Docker (如未安装)
2. 自动安装 Docker Compose (如未安装)
3. 克隆或同步仓库代码
4. 配置环境变量
5. 启动 Docker 服务
6. 验证部署状态

### 2.2 本地已有仓库

```bash
cd /path/to/basjoo
sudo sh install-deploy.sh
```

### 2.3 Docker Compose 部署

#### 开发环境

```bash
docker compose --profile dev up -d
```

**开发服务**:
- `backend-dev`: 后端开发服务 (挂载代码)
- `frontend-dev`: 前端开发服务 (热重载)
- `nginx`: 反向代理

#### 生产环境

```bash
docker compose --profile prod up -d
```

**生产服务**:
- `backend`: 后端生产服务
- `frontend`: 前端生产服务
- `nginx`: 反向代理

## 3. 环境配置

### 3.1 环境变量

创建 `.env` 文件或设置环境变量:

```bash
# 数据库
DATABASE_URL=sqlite:///./data/basjoo.db

# Redis
REDIS_URL=redis://redis:6379/0

# Qdrant
QDRANT_URL=http://qdrant:6333
QDRANT_API_KEY=  # 可选，有密码时设置

# JWT 密钥 (自动生成)
SECRET_KEY=your-secret-key-change-in-production
SECRET_KEY_FILE=/app/data/.secret_key

# 默认 Agent ID (自动生成)
DEFAULT_AGENT_ID=
AGENT_ID_FILE=/app/data/.agent_id

# API 密钥
DEEPSEEK_API_KEY=sk-xxx  # 可选，用于默认 Agent
JINA_API_KEY=  # 可选

# CORS 配置
ALLOWED_ORIGINS=*  # 生产环境建议指定域名

# 限流配置
RATE_LIMIT_PER_MINUTE=1000
RATE_LIMIT_BURST_SIZE=200

# 日志级别
LOG_LEVEL=info

# Scrapling 服务
SCRAPLING_SERVICE_URL=http://scrapling-service:8001
```

### 3.2 加密配置

```bash
# 加密密钥 (自动生成)
ENCRYPTION_KEY=
ENCRYPTION_KEY_FILE=/app/data/.encryption_key

# 生产环境必须设置
REQUIRE_SECRET_KEY=true
```

### 3.3 服务器域名

```bash
# 设置服务器域名 (可选)
SERVER_DOMAIN=example.com
```

## 4. Docker 服务

### 4.1 服务架构

```
┌─────────────────────────────────────────────────────────┐
│                     Nginx (Port 80/443)                  │
│              (反向代理 + SSL + 请求限制)                   │
└─────────────────────┬───────────────────────────────────┘
                      │
         ┌────────────┴────────────┐
         │                         │
         ▼                         ▼
┌─────────────────┐     ┌─────────────────┐
│  Frontend (3000)│     │  Backend (8000)  │
│  Next.js        │     │  FastAPI         │
└─────────────────┘     └────────┬──────────┘
                                 │
         ┌───────────────────────┼───────────────────────┐
         │                       │                       │
         ▼                       ▼                       ▼
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  Qdrant (6333)  │     │  Redis (6379)   │     │  PostgreSQL     │
│  向量数据库     │     │  缓存/限流      │     │  (5432)         │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 4.2 端口映射

| 服务 | 内部端口 | 外部端口 | 说明 |
|------|----------|----------|------|
| nginx | 80 | 80/443 | HTTP/HTTPS |
| backend | 8000 | - | 后端 API |
| frontend | 3000 | - | 前端开发 |
| qdrant | 6333 | 6333 | Qdrant Dashboard |
| redis | 6379 | 6379 | Redis (可限制) |
| postgres | 5432 | 5432 | PostgreSQL (可限制) |

## 5. HTTPS 配置

### 5.1 自动 HTTPS

将证书和密钥文件放入 `ssl/` 目录:

```bash
mkdir -p ssl
cp your-cert.crt ssl/certificate.crt
cp your-key.key ssl/private.key
```

重启服务后自动启用 HTTPS:

```bash
docker compose --profile prod restart nginx
```

### 5.2 HTTP 重定向

启用 HTTPS 后，HTTP 请求自动重定向到 HTTPS。

## 6. 数据持久化

### 6.1 持久化卷

| 卷名 | 路径 | 说明 |
|------|------|------|
| backend-data | /app/data | 数据库、密钥文件 |
| redis-data | /data | Redis 数据 |
| postgres-data | /var/lib/postgresql/data | PostgreSQL 数据 |

### 6.2 备份数据

```bash
# 备份数据库
docker compose exec backend tar -czf /app/data/backup.tar.gz -C /app/data .

# 备份到宿主机
docker compose exec backend cat /app/data/basjoo.db > backup.db

# 备份完整数据目录
docker compose exec backend tar -czf - -C /app data | tar -xzf - -C /path/to/backup
```

### 6.3 恢复数据

```bash
# 停止服务
docker compose --profile prod down

# 恢复数据库
docker compose exec -T backend sh -c 'cat > /app/data/basjoo.db' < backup.db

# 重启服务
docker compose --profile prod up -d
```

## 7. 更新部署

### 7.1 一键更新

```bash
cd /path/to/basjoo
git pull origin main  # 或指定分支
sudo sh install-deploy.sh
```

### 7.2 手动更新

```bash
# 拉取最新代码
git pull origin main

# 重建并重启服务
docker compose --profile prod up -d --build

# 查看日志
docker compose --profile prod logs -f backend
```

### 7.3 保留 Widget Agent ID

在迁移时，确保保留原有的 Agent ID:

1. 备份 `/app/data/.agent_id`
2. 在新服务器设置 `DEFAULT_AGENT_ID=<old_agent_id>`
3. 启动服务

## 8. 故障排查

### 8.1 查看日志

```bash
# 查看所有服务日志
docker compose logs -f

# 查看特定服务
docker compose logs -f backend
docker compose logs -f frontend

# 查看最近日志
docker compose logs --tail=100 backend
```

### 8.2 健康检查

```bash
# 后端健康检查
curl http://localhost:8000/health

# Qdrant 健康检查
curl http://localhost:6333/health

# Redis 健康检查
docker compose exec redis redis-cli ping
```

### 8.3 常见问题

#### 数据库锁定

```bash
# 重启后端
docker compose --profile prod restart backend
```

#### 内存不足

```bash
# 检查内存使用
docker stats

# 增加 Swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

#### 磁盘空间不足

```bash
# 清理 Docker
docker system prune -a

# 清理日志
docker compose logs --tail=0 > /dev/null  # 刷新日志
find /var/lib/docker/containers -name '*.json' -exec truncate -s 0 {} \;
```

## 10. 卸载

```bash
# 停止服务
docker compose --profile prod down

# 删除卷 (谨慎! 会丢失数据)
docker volume rm basjoo_backend-data basjoo_redis-data basjoo_postgres-data

# 删除代码目录
rm -rf /path/to/basjoo
```
