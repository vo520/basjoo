# 后端架构文档

## 1. 架构概览

Basjoo 后端采用 **FastAPI** 框架构建，核心架构分为以下层次：

```
┌─────────────────────────────────────────────────────────┐
│                    API Gateway Layer                      │
│         (CORS, Rate Limit, I18n Middleware)              │
├─────────────────────────────────────────────────────────┤
│                     Router Layer                         │
│   /api/admin/*  │  /api/v1/*  │  /sdk.js  │  /health   │
├─────────────────────────────────────────────────────────┤
│                    Service Layer                         │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ │
│  │ LLM Svc  │ │  KB Svc  │ │ URL Svc  │ │ Auth Svc │ │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ │
├─────────────────────────────────────────────────────────┤
│                    Data Layer                            │
│     SQLite (App Data)  │  Qdrant (Vectors)  │  Redis   │
└─────────────────────────────────────────────────────────┘
```

## 2. 核心模块

### 2.1 入口文件 (`main.py`)

**职责**:
- FastAPI 应用初始化
- 中间件配置
- 路由挂载
- 调度器启动

**关键配置**:
```python
app = FastAPI(
    title="Basjoo",
    description="智能体（agent）系统",
    version="1.0.0",
    lifespan=lifespan,
)
```

**中间件顺序**:
1. CORSMiddleware
2. I18nMiddleware
3. RateLimitMiddleware
4. 日志中间件 (请求体大小检查)

### 2.2 认证模块 (`api/endpoints/auth.py`)

**端点**:
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/admin/register` | 管理员注册 |
| POST | `/api/admin/login` | 管理员登录 |
| GET | `/api/admin/me` | 获取当前用户 |

**权限装饰器**:
- `get_current_admin`: 获取当前认证用户
- `require_admin_or_super_admin`: 管理员或超级管理员
- `require_super_admin`: 仅超级管理员
- `require_chat_operator`: 聊天操作员

### 2.3 v1 API 模块 (`api/v1/endpoints.py`)

**分类端点**:

#### Chat API
- `POST /api/v1/chat` - 聊天请求
- `POST /api/v1/chat/stream` - 流式聊天 (SSE)
- `GET /api/v1/chat/messages` - 获取聊天消息
- `POST /api/v1/chat/:session_id/takeover` - 接管会话

#### Agent API
- `GET /api/v1/agent:default` - 获取默认 Agent
- `GET /api/v1/agent:config` - 获取 Agent 配置
- `POST /api/v1/agents` - 创建 Agent
- `GET /api/v1/agents` - 列出 Agents
- `PUT /api/v1/agents/:id` - 更新 Agent
- `DELETE /api/v1/agents/:id` - 删除 Agent

#### URL API
- `POST /api/v1/urls:create` - 创建 URL
- `GET /api/v1/urls:list` - 列出 URLs
- `POST /api/v1/urls:refetch` - 重新抓取
- `DELETE /api/v1/urls/:id` - 删除 URL
- `POST /api/v1/urls:clear_all` - 清空所有 URLs

#### File API
- `POST /api/v1/files:upload` - 上传文件
- `GET /api/v1/files:list` - 列出文件
- `DELETE /api/v1/files/:id` - 删除文件

#### Index API
- `POST /api/v1/index:rebuild` - 重建索引
- `GET /api/v1/index:status` - 索引状态
- `GET /api/v1/index:info` - 索引信息

#### Context API
- `POST /api/v1/contexts` - 检索上下文

#### Session API
- `GET /api/v1/sessions` - 列出会话
- `PUT /api/v1/sessions/:id/close` - 关闭会话

#### Quota API
- `GET /api/v1/quota` - 获取配额信息

### 2.4 服务层 (`services/`)

#### LLM Service (`llm_service.py`)

**职责**: 封装 LLM 调用，支持多种提供商

**类层次**:
```
BaseLLM (ABC)
├── OpenAICompatibleLLM
├── OpenAINativeLLM
├── GoogleLLM
├── AnthropicLLM
└── MockLLM (测试用)
```

**错误分类**:
- `APIKeyInvalidError`: API 密钥无效
- `APIKeyMissingError`: API 密钥缺失
- `ProviderRateLimitedError`: 提供商限流
- `ProviderUnavailableError`: 提供商不可用
- `ModelNotFoundError`: 模型未找到

#### KB Service (`kb_service.py`)

**职责**: 知识库 CRUD 操作

**核心方法**:
- `create_knowledge_base()`: 创建知识库
- `list_knowledge_bases()`: 列出知识库
- `get_knowledge_base()`: 获取知识库
- `update_kb_config()`: 更新配置
- `delete_knowledge_base()`: 删除知识库

#### Qdrant Service (`qdrant_service.py`)

**职责**: Qdrant 向量数据库操作

**核心方法**:
- `ensure_collection()`: 幂等创建 Collection
- `batch_upsert_points()`: 批量写入向量
- `delete_points_by_doc_id()`: 按文档 ID 删除
- `search_kb()`: 知识库检索

**向量维度映射**:
| 模型 | 维度 |
|------|------|
| BGE-M3 | 1024 |
| Jina embeddings | 1024 |
| text-embedding-3-large | 3072 |
| text-embedding-3-small | 1536 |

#### URL Service (`url_service.py`)

**职责**: URL 知识源管理

**核心方法**:
- `create_urls()`: 创建 URL
- `list_urls()`: 列出 URLs
- `delete_url()`: 删除 URL
- `clear_all_urls()`: 清空所有 URLs

#### KB Document Service (`kb_document_endpoints.py`)

**职责**: 文档上传和处理

**端点**:
- `POST /api/v1/tenants/{tenant_id}/knowledge_bases/{kb_id}/documents` - 上传文档
- `GET /api/v1/tenants/{tenant_id}/knowledge_bases/{kb_id}/documents` - 列出文档
- `DELETE /api/v1/tenants/{tenant_id}/knowledge_bases/{kb_id}/documents/{doc_id}` - 删除文档

#### Retrieval Service (`kb_retrieval_service.py`)

**职责**: 知识检索

**核心方法**:
- `retrieve()`: 检索相关片段

**流程**:
1. 获取 Agent 绑定的 KB
2. 嵌入查询文本
3. Qdrant 相似度搜索
4. 阈值过滤
5. 返回 Top-K 结果

#### Auth Service (`auth_service.py`)

**职责**: 认证授权逻辑

**核心方法**:
- `verify_password()`: 验证密码
- `get_password_hash()`: 密码哈希
- `create_access_token()`: 创建 JWT Token
- `verify_token()`: 验证 Token

#### Redis Service (`redis_service.py`)

**职责**: Redis 连接和操作

**核心方法**:
- `get_redis()`: 获取 Redis 连接
- `close_redis()`: 关闭连接
- `health_check()`: 健康检查

#### Scheduler Service (`scheduler.py`)

**调度器**:

1. **URLFetchScheduler**
   - 每小时检查需要更新的 URLs
   - 按 `url_fetch_interval_days` 间隔重新抓取

2. **HistoryCleanupScheduler**
   - 每天凌晨 3 点执行
   - 删除超过 `history_days` 的会话

3. **SessionAutoCloseScheduler**
   - 每 5 分钟执行
   - 关闭 30 分钟无活动的会话

4. **AgentPurgeScheduler**
   - 每 24 小时执行
   - 永久删除软删除的 Agent

### 2.5 安全模块

#### SSRF 防护 (`services/url_safety.py`)

**阻止的类型**:
- localhost / 127.0.0.1
- 内网 IP (RFC1918)
- Link-local 地址
- 多播地址
- 未指定地址
- 直接 IP 文字

**白名单**:
- IANA 基准测试范围 `198.18.0.0/15`

#### 加密模块 (`core/encryption.py`)

**算法**: Fernet (AES-128-CBC + HMAC)

**用途**:
- Agent API 密钥加密存储
- 环境变量 `ENCRYPTION_KEY` 配置

### 2.6 中间件 (`middleware/`)

#### Rate Limit Middleware (`rate_limit.py`)

**功能**:
- 基于 Redis 的请求限流
- 支持内存回退 (Redis 不可用时)
- 共享 `apply_cors_headers()` 辅助函数

**配置**:
- `RATE_LIMIT_PER_MINUTE`: 每分钟请求数
- `RATE_LIMIT_BURST_SIZE`: 突发大小

#### I18n Middleware (`i18n/core.py`)

**功能**:
- 自动检测请求语言
- 注入翻译函数到请求上下文

### 2.7 数据模型 (`models.py`)

#### 核心实体

```
Workspace (工作空间)
├── Agent (智能体)
│   ├── URLSource (URL 知识源)
│   ├── KnowledgeFile (文件知识源)
│   ├── ChatSession (聊天会话)
│   │   └── ChatMessage (聊天消息)
│   └── AgentMember (成员关系)
├── WorkspaceQuota (配额)
└── AdminUser (管理员)
    └── AgentMember

Tenant (租户)
└── KnowledgeBase (知识库)
    ├── KbDocument (文档)
    │   └── KbChunk (片段)
    └── Agent (关联)
```

#### 模型关系

| 模型 | 关系 | 说明 |
|------|------|------|
| Workspace | 1:N Agent | 工作空间包含多个 Agent |
| Agent | 1:N URLSource | Agent 管理多个 URL |
| Agent | 1:N KnowledgeFile | Agent 管理多个文件 |
| Agent | 1:N ChatSession | Agent 产生多个会话 |
| Agent | N:N AdminUser | 通过 AgentMember 关联 |
| Agent | 1:1 KnowledgeBase | Agent 可绑定一个 KB |
| KnowledgeBase | 1:N KbDocument | KB 包含多个文档 |
| KbDocument | 1:N KbChunk | 文档包含多个片段 |

### 2.8 配置管理 (`config.py`)

**配置来源**:
1. 环境变量
2. `.env` 文件
3. 默认值

**关键配置**:
```python
# 数据库
database_url: str = "sqlite:///./data/basjoo.db"

# Redis
redis_url: str = "redis://redis:6379/0"

# Qdrant
qdrant_url: str = "http://localhost:6333"

# JWT
secret_key: str = ""  # 自动生成并持久化
algorithm: str = "HS256"

# 限流
rate_limit_per_minute: int = 1000
rate_limit_burst_size: int = 200

# CORS
allowed_origins: str = "*"
```

## 4. 错误处理

### HTTP 状态码

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误 |
| 401 | 未认证 |
| 403 | 权限不足 |
| 404 | 资源未找到 |
| 410 | 资源已删除 |
| 413 | 请求体过大 |
| 429 | 请求过于频繁 |
| 500 | 服务器内部错误 |

### 错误响应格式

```json
{
  "detail": "错误描述"
}
```

### LLM 错误映射

| 原始错误 | 分类错误码 |
|----------|------------|
| 401 Invalid API Key | `API_KEY_INVALID` |
| 429 Rate Limit | `PROVIDER_RATE_LIMITED` |
| 404 Model Not Found | `MODEL_NOT_FOUND` |
| Timeout/Connection | `PROVIDER_UNAVAILABLE` |

## 6. 扩展性

### 6.1 新增 LLM 提供商

1. 在 `llm_service.py` 添加新类继承 `BaseLLM`
2. 在 `provider_type` 枚举中添加新类型
3. 在前端 UI 添加提供商选项

### 6.2 新增知识源类型

1. 在 `models.py` 添加新模型
2. 在 `services/` 添加处理逻辑
3. 添加相应 API 端点

### 6.3 新增 Embedding 提供商

1. 在 `document_parser.py` 添加嵌入逻辑
2. 更新前端配置选项
