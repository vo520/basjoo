# 开发指南

## 1. 开发环境搭建

### 1.1 前置要求

- Python 3.10+
- Node.js 18+
- Docker & Docker Compose

### 1.2 克隆仓库

```bash
git clone https://github.com/haoyiyin/basjoo.git
cd basjoo
```

### 1.3 Docker 开发环境

```bash
# 启动开发服务
docker compose --profile dev up -d

# 查看日志
docker compose logs -f backend-dev frontend-dev
```

### 1.4 本地开发

#### 后端

```bash
cd backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动服务
python3 main.py
```

#### 前端

```bash
cd frontend-nextjs

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

#### Widget

```bash
cd widget

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

## 3. 代码规范

### 3.1 Python (后端)

- 使用 4 空格缩进
- 函数和变量使用 snake_case
- 类名使用 PascalCase
- 常量使用 UPPER_SNAKE_CASE
- 所有公共 API 添加 docstring

```python
def get_user_by_id(user_id: int) -> User | None:
    """
    根据用户 ID 获取用户信息。
    
    Args:
        user_id: 用户 ID
        
    Returns:
        User 对象或 None
    """
    pass
```

### 3.2 TypeScript (前端)

- 使用 2 空格缩进
- 组件和类使用 PascalCase
- 函数和变量使用 camelCase
- 常量使用 UPPER_SNAKE_CASE
- 所有组件添加类型定义

```typescript
interface UserProps {
  userId: string;
  onUpdate?: (user: User) => void;
}

export function UserProfile({ userId, onUpdate }: UserProps) {
  // ...
}
```

### 3.3 Git 提交规范

使用 Conventional Commits:

```
<type>(<scope>): <subject>

[optional body]

[optional footer]
```

**type**:
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档变更
- `style`: 代码格式
- `refactor`: 重构
- `test`: 测试
- `chore`: 构建/工具

**示例**:

```
feat(chat): add streaming response support

fix(url): handle SSRF protection for IPv6 addresses

docs(api): update endpoint documentation
```

## 5. API 开发

### 5.1 添加新端点

1. 在 `schemas.py` 添加请求/响应模型

```python
class NewFeatureRequest(BaseModel):
    name: str = Field(..., description="名称")
    value: int = Field(0, ge=0, description="值")

class NewFeatureResponse(BaseModel):
    id: str
    name: str
    value: int
```

2. 在 `endpoints.py` 添加路由

```python
@router.post("/new-feature", response_model=NewFeatureResponse)
async def create_new_feature(
    request: NewFeatureRequest,
    db: AsyncSession = Depends(get_db),
    current_user: AdminUser = Depends(get_current_admin),
):
    # 实现逻辑
    return NewFeatureResponse(id="xxx", name=request.name, value=request.value)
```

### 5.2 添加新服务

1. 在 `services/` 创建服务文件

```python
# services/new_service.py
class NewService:
    def __init__(self):
        pass
    
    async def do_something(self, param: str) -> str:
        return f"Processed: {param}"
```

2. 在 `services/__init__.py` 导出

```python
from .new_service import NewService
```

### 5.3 添加新数据模型

1. 在 `models.py` 添加模型

```python
class NewModel(Base):
    __tablename__ = "new_models"
    
    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)
    created_at = Column(DateTime, server_default=func.now())
```

2. 运行迁移


## 10. 性能优化

### 10.1 后端优化

- 使用索引优化查询
- 批量处理减少数据库往返
- 使用缓存减少重复计算
- 异步 I/O 提高并发

### 10.2 前端优化

- 组件懒加载
- 减少不必要的渲染
- 使用 React.memo
- 优化 bundle 大小

### 10.3 数据库优化

```python
# 使用索引
Index("ix_table_column", "column")

# 使用批量插入
session.bulk_insert_mappings(Model, list_of_dicts)

# 使用查询优化
query = select(Model).options(joinedload(Model.relation))
```
